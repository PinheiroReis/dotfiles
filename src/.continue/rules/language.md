# Language Rules

## Communication Language
- **Always communicate with the user in Portuguese (PT-BR)**
- All explanations, discussions, and direct interactions should be in Portuguese

## Code and Technical Documentation
- **All code must be written in English**
  - Variable names
  - Function names
  - Class names
  - Constants
  - Any code identifiers

- **All documentation must be in English**
  - README files
  - API documentation
  - Technical guides
  - Inline comments
  - Docstrings
  - Commit messages
  - Pull request descriptions

## Exceptions
- The user may explicitly request content in a different language
- When requested, override these default rules accordingly

## Summary
- **User interaction**: Portuguese (PT-BR)
- **Code & Documentation**: English
- **Override**: Only when explicitly requested by the user
